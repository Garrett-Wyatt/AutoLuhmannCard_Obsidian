

# 🚀 Auto Luhmann Card

> **一套基于系统工程思维的多模态知识解构与自动化挂载方案。**

本仓库旨在解决信息爆炸时代“收藏不看”的痛点，通过标准化协议将碎片链接（视频、推特、长文）转化为具备逻辑关联的原子化卡片。

---

## 📁 目录结构说明

根据本地库结构，各模块职能如下：

* **`00_System`**：系统底座。存放用于自动化处理的脚本（JavaScript/Templater）及环境配置文件。
* **`Get_Info`**：信息注入总线。全平台多模态信息的原始采集点（Inbox），负责信息的“标准化脱水”前置处理。
* **`Zettelkasten`**：原子知识库。存放执行完“原子化协议”后的正式卢曼卡片，实现知识的非线性生长。
* **`模板`**：协议定义中心。包含 Text Generator 的 AI 解构模板及 Obsidian 核心样式模板。

---

## 🛠️ 核心工作流

1. **输入**：将碎片链接或内容投喂至 `Get_Info`。
2. **解构**：调用 `模板` 中的 AI 算子，对信息进行强制原子化提炼。
3. **挂载**：系统自动计算 `up_link` 与 `origin_link`，确立知识位标。
4. **归档**：处理后的成品卡片自动存入 `Zettelkasten`。
